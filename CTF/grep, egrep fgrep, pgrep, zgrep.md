Блок 1. Что такое grep  
grep нужен для поиска слов по шаблонам внутри файлов или вывода других команд.  
Основная идея: grep получает входные данные (из файла или через пайп) и фильтрует строки согласно шаблону.

---

Блок 2. Базовые примеры  
Пример поиска слова в файле:  
grep -i "bebe" file.txt  
Здесь ключ -i означает игнор регистра, то есть найдёт "bebe", "BEBE", "BeBe" и т.д.

Поиск в выводе другой команды через пайп:  
ls | grep Documents  
Если grep ничего не вернул, значит нет строки, содержащей "Documents".

---

Блок 3. Основные ключи  
Ключ -i — игнор регистра.  
Ключ -e — позволяет указать несколько шаблонов.  
Пример с несколькими шаблонами:  
grep -e "bebe" -e "blabla" file.txt

---

Блок 4. Родственные утилиты и аналоги  
egrep, fgrep, pgrep, zgrep — вариации grep с другим поведением по умолчанию.  
Пример:  
egrep "bebe" file.txt = grep -E "bebe" file.txt (регулярки в расширенном синтаксисе)  
fgrep — эквивалент grep -F (поиск без регулярных выражений).  
pgrep sshd тоже самое концептуально, что ps e | grep sshd, только pgrep фильтрует процессы напрямую.  
zgrep — работает с архивами .gz.

---

Блок 5. Рекурсивный поиск  
grep -r "picoCTF{" /home — рекурсивный поиск по директориям.  
rgrep "picoCTF{" /home — делает по сути то же самое.
